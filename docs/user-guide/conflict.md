# 冲突解决指南

## 什么是同步冲突？

当您在多个设备上同时修改同一个技能时，可能会发生同步冲突。这意味着本地版本和 iCloud 版本的内容不同，系统无法自动决定保留哪个版本。

## 如何识别冲突？

当存在冲突时，Library 页面顶部会显示黄色警告提示：

> **N 个技能存在同步冲突**

点击"查看并解决"按钮可打开冲突解决弹窗。

## 冲突解决选项

对于每个冲突，您可以选择以下选项：

| 选项 | 说明 | 适用场景 |
|------|------|----------|
| **保留本地版本** | 使用当前设备上的版本，覆盖其他设备的修改 | 您确定本地修改是最新且正确的 |
| **保留远程版本** | 使用 iCloud 上的版本，丢弃当前设备的修改 | 其他设备上的修改更重要 |
| **保留两者** | 创建两个副本，您可以稍后手动合并 | 两个版本都有重要内容需要合并 |

## 冲突类型

| 类型 | 说明 |
|------|------|
| 内容冲突 | `SKILL.md` 文件内容不同 |
| 元数据冲突 | `skill.json` 文件内容不同 |
| 删除冲突 | 一边删除了技能，另一边修改了它 |

## 最佳实践

1. **定期同步**: 减少离线操作时间，尽早发现冲突
2. **查看差异**: 解决冲突前，先查看两个版本的差异
3. **合并内容**: 选择"保留两者"后，记得手动合并重要内容
4. **备份重要技能**: 在解决复杂冲突前，可以导出技能作为备份

## 常见问题

### 为什么会出现冲突？

iCloud 同步是异步的，当您在离线时修改技能，其他设备也修改了同一技能，就会产生冲突。

### 冲突会丢失数据吗？

不会。冲突检测的目的就是防止数据丢失。在您手动解决之前，两个版本都会保留。

### 选择"保留两者"后如何合并？

系统会创建一个以 `_from_icloud_时间戳` 结尾的副本。您可以打开两个版本，手动复制需要的内容到主版本，然后删除副本。
